# 01. Requirements

## 과제 목표

- PostgreSQL 기반 데이터베이스 웹 서비스를 구현한다.
- 스터디 모집 서비스를 주제로 CRUD API와 트랜잭션 처리를 설계한다.
- `pgbench`로 격리 수준별 동시성 동작과 성능 차이를 비교한다.

## 핵심 시나리오

1. 사용자가 회원으로 등록한다.
2. 사용자가 스터디를 생성한다.
3. 다른 사용자가 스터디에 참여한다.
4. 정원을 초과하면 참여가 실패해야 한다.
5. 동일 기능을 `Read Committed`, `Serializable`로 비교한다.

## 핵심 기능 목록

- 사용자 생성 / 목록 조회 / 상세 조회
- 스터디 생성 / 목록 조회 / 상세 조회 / 수정 / 삭제
- 스터디 참여 / 참여 취소

## 최종 제출물 정리

- GitHub 저장소
- 발표용 PPT
- 선택 사항: 서비스 배포 주소
